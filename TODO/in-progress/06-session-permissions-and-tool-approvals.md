# MVP-06：Session 权限与工具审批

- 状态：未开始
- 优先级：P0
- 前置任务：MVP-05
- 后续任务：MVP-07

## 目标

让每个 Session 保存自己的 OMP 审批模式，并在当前 OMP v17.0.6 RPC 能力上提供紧凑、可恢复的工具审批交互。

## 明确不做

- 不修改或 Fork OMP v17.0.6。
- 不把权限解释为 Linux 文件系统沙箱或 Electron 系统权限。
- 不通过模糊标题匹配识别工具审批。
- 不实现每个工具的 `allow`、`prompt`、`deny` 编辑器。
- 不增加权限历史数据库，不复制 OMP Session 历史。
- 不把完整审批 `title`、命令或写入 Content 写入 Renderer 状态、诊断日志或持久化文件。

## 任务清单

### Session 权限配置

- [ ] 在输入框下方、模型和 Thinking Level 旁增加权限按钮，显示`权限：严格`、`权限：标准`或`权限：全部允许`。
- [ ] 权限 Popover 只显示“严格”“标准”“全部允许”三个中文名称；打开后聚焦当前权限，选择后立即关闭，不增加“保存”按钮。
- [ ] 权限按钮和 Popover 支持 Tab 聚焦、Enter、方向键和 Esc，不增加全局快捷键。
- [ ] 固定映射`严格 → always-ask`、`标准 → write`、`全部允许 → yolo`；启动参数、日志和错误详情使用英文原值。
- [ ] 未选中 Session 时显示禁用的`权限：全部允许`；Session 列表不显示权限标签。
- [ ] 新建 Session 默认使用并保存“全部允许”；发送第一条消息前修改权限时立即保存，但发送首条消息时才启动 OMP。
- [ ] Main 按 Workspace ID 和 Session ID 将权限写入 Desktop 数据目录的配置文件，使用原子替换，不修改项目 `.omp/config.yml`，不增加数据库。
- [ ] 只接受 `always-ask`、`write`、`yolo`；缺失或未知值按 `yolo` 打开，记录诊断日志并补存正确值。
- [ ] 新 Session 权限保存失败时恢复上一次已保存的值，在输入框附近显示“权限保存失败”，不启动或重启 Runtime。
- [ ] 重复选择当前权限时只关闭 Popover，不写配置、不重启 Runtime、不显示转圈。
- [ ] 三种权限使用相同的普通按钮样式，不为“全部允许”增加警告色或风险提示。
- [ ] Session 删除成功后清理权限、文字草稿、附件和历史缓存，删除失败时全部保留；归档保留权限和文字草稿，可以释放附件和历史缓存。
- [ ] 文字草稿继续复用现有按 Workspace 和 Session 区分的 `localStorage`；附件只在当前 Desktop 运行期间按 Session 保留。

### Runtime 启动与 Session 切换

- [ ] 在 `RuntimeSnapshot` 和受控 IPC 中加入当前 Session 保存的权限模式。
- [ ] 新建、打开、切换或异常恢复 Session 时读取其权限，Main 通过固定枚举和参数数组传入 `--approval-mode <mode>`，不经过 Shell。
- [ ] 目标 Session 与当前 Runtime 权限相同时复用 Runtime；不同时重启 Runtime，再恢复目标 Session。
- [ ] 权限相同且不重启时，权限按钮不转圈；Session 恢复超过 150 毫秒时只在聊天区域显示加载状态。
- [ ] 重启时不弹窗、不遮挡聊天页面，输入框保持可编辑；权限按钮立即显示目标权限并带小转圈，发送、权限修改和 Session 切换暂时禁用，不提供取消。
- [ ] Renderer 只缓存当前 Session 和最近离开的 1 个 Session 的历史展示投影；不缓存 RPC 原始事件、Tool 完整输出、图片 Base64、二进制、动画或进度，也不跨 Desktop 重启保存。
- [ ] 有历史缓存时立即显示目标 Session，OMP 恢复后按消息 ID 以真实历史校正；没有缓存时等待 OMP，超过 150 毫秒后在聊天区域显示“正在打开会话…”。
- [ ] 修改当前 Session 权限后重启并恢复；新 Runtime 就绪后才保存新值。
- [ ] 新权限启动失败时恢复原值并按原权限重新启动；恢复成功只显示“权限切换失败，已恢复为「当前权限」”。
- [ ] 权限切换成功后只结束转圈并保留目标权限，不显示成功提示。
- [ ] Session 打开失败时停留在目标 Session，并允许选择另一权限重试；新选择仍在启动成功后才保存。
- [ ] 有历史缓存时在输入框附近显示启动错误；没有缓存时在聊天区域显示“会话打开失败”和“重试”，不重复显示。
- [ ] 新旧权限均启动失败时保留历史、草稿和附件，显示“重试”和“查看错误”；重试仍使用原权限。
- [ ] 沿用 Runtime Supervisor 的 15 秒启动超时；Session 切换失败后不自动重试。
- [ ] 错误详情显示退出码和脱敏后的最后 20 KB `stderr`，支持复制，不显示环境变量、模型密钥或代理密码。
- [ ] 重启期间允许直接关闭 Desktop；未成功的权限修改不保存。
- [ ] 切换到 `yolo` 不显示风险确认。
- [ ] Runtime 执行中、Follow-up 队列非空或等待 Interaction 时禁用权限修改和 Session 切换；用户需要先 Stop。

### OMP 版本与审批识别

- [ ] 启动时读取 OMP 版本；只有 v17.0.6 启用当前工具审批适配器。
- [ ] Main 只把`method: select`、精确的 `["Approve", "Deny"]` 和首行`Allow tool: `同时匹配的请求识别为工具审批，不做模糊匹配。
- [ ] 未知 OMP 版本或格式不匹配时回退为 MVP-04 的通用 Interaction，不进入权限合并区、不自动选择，并记录兼容诊断日志。
- [ ] 未知版本每次 Runtime 启动最多显示一次“当前 OMP 版本未验证，权限确认使用兼容模式”，不阻塞聊天。
- [ ] 普通 `select` 带 OMP 超时时显示剩余时间，到期移除按钮并显示“已超时”；没有超时时等待用户选择或 OMP 取消。
- [ ] 当前模式为“全部允许”时仍处理 OMP 实际发出的 `select`，不能按 Desktop 模式隐藏或跳过。
- [ ] 后续 OMP 版本必须先增加对应 RPC fixture 和识别测试，才能启用紧凑审批与自动允许。

### 工具审批 UI

- [ ] 工具审批按钮显示为“拒绝”“允许”，回传值仍为 `Deny`、`Approve`；默认焦点放在“允许”，Enter 执行当前焦点，Esc 拒绝。
- [ ] 工具审批默认等待 30 秒；OMP 提供更短超时时采用较短值，提供更长时间仍使用 30 秒。
- [ ] Main 保存绝对截止时间；窗口失焦、最小化或系统睡眠不重置，唤醒时已过期则立即自动允许。
- [ ] 第一个请求到达后等待 100 毫秒收集同批请求；OMP 期限不足时缩短或跳过等待。
- [ ] 单个请求不显示计数和批量按钮；追加第二个请求时原区域变为合并确认区，保持焦点、截止时间和页面位置。
- [ ] 多个请求合并到一个确认区，每行提供“拒绝”“允许”，底部固定“全部拒绝”“全部允许”；默认焦点放在“全部允许”，Esc 全部拒绝。
- [ ] 顶部显示`待确认 N / 总数`；最多直接显示 5 行，更多内容在区域内滚动。
- [ ] 新请求追加到末尾，不抢焦点、不强制滚动，并沿用当前剩余时间；处理单项和追加请求都不重置截止时间。
- [ ] 单项决定立即回传，不等待整组；批量决定只处理点击瞬间已经进入确认区的未处理请求。
- [ ] 到期只自动允许尚未处理的请求。
- [ ] 已处理行保留到整组完成，显示“已允许”“已自动允许”“已拒绝”“已取消”或“请求已失效”；全部完成或取消后确认区立即消失。
- [ ] 合并确认区放在当前 Run 过程区域底部；到达时自动展开并滚动到确认区，用户仍可再次折叠。
- [ ] 折叠摘要显示“等待操作 · `N` 秒后自动允许”；自动允许后保持折叠并恢复“进行中”，新权限请求到达时再次展开。
- [ ] 窗口在后台或最小化时不发送权限系统通知。

### 紧凑摘要

- [ ] Main 从审批 `title` 提取中文工具类别和主要目标，只向 Renderer 发送摘要、请求 ID、状态和截止时间。
- [ ] 权限行只占一行，按可用宽度显示，超出使用省略号，不设置固定字符上限，不显示写入 Content。
- [ ] Workspace 内路径显示相对路径，外部路径显示绝对路径；无法可靠处理时保留 OMP 原文。
- [ ] URL 隐藏协议、用户信息和查询参数，只显示域名与路径。
- [ ] 命令在 Workspace 根目录执行时只显示命令，在子目录或 Workspace 外执行时显示工作目录。
- [ ] 多文件请求显示第一个路径和文件总数；未知工具保留原始名称。
- [ ] `title` 解析只用于显示摘要，不能用于判断权限或关联 Tool Call。
- [ ] 完整审批 `title` 只在 Main 内存中短暂保留，不进入 Renderer、日志或持久化数据。

### 状态、取消与恢复

- [ ] OMP `cancel` 到达时停止对应倒计时，显示“已取消”并禁用该行，不能再自动允许。
- [ ] 用户 Stop 时取消全部待处理请求和倒计时，不自动允许剩余请求；关闭 Desktop 时结束 Runtime，不保存或重放待处理请求。
- [ ] Main 保证每个请求只完成一次；用户操作与超时竞争时采用先处理的结果。
- [ ] 相同请求 ID 的重复事件只保留第一条，不增加行、不重置倒计时、不重复回传，并记录诊断警告。
- [ ] 请求完成或取消后的迟到操作直接忽略；Renderer 使用 Main 的最终状态纠正界面。
- [ ] 回传时 OMP 已断开则显示“请求已失效”，清除倒计时，不向新 Runtime 重放。
- [ ] 待确认请求、状态和截止时间进入 `RuntimeSnapshot`；Renderer 重载后恢复当前确认区，不重置倒计时或显示过期按钮。
- [ ] 权限诊断日志只记录 Session ID、RPC 请求 ID、结果和处理时间，不记录完整命令、Content 或审批 `title`。
- [ ] 审批方式只在当前运行期间显示；重开 Session 后只恢复 Tool Call 最终状态，不增加权限历史数据库。

### 测试

- [ ] 测试三种权限的中英文映射、默认“全部允许”、按 Session 隔离保存、原子写入、未知值回退和固定参数数组。
- [ ] 测试 Session 删除、归档、缺失或无效值补存 `yolo`、草稿与附件清理规则。
- [ ] 测试无 Session 时的禁用按钮、Session 列表不显示权限、Popover 初始焦点和重复选择当前权限。
- [ ] 测试新 Session 修改权限后立即保存但延迟启动 Runtime，以及保存失败时恢复原值。
- [ ] 测试相同权限复用 Runtime，不同权限重启并恢复目标 Session；异常重启继续使用当前 Session 权限。
- [ ] 测试重启期间按钮显示目标权限，成功后不提示，失败后恢复显示原权限。
- [ ] 测试权限修改成功后才保存新值；失败时恢复原值并按原权限启动；新旧权限均失败时进入 Runtime 错误状态。
- [ ] 测试 15 秒启动超时、不自动重试、错误脱敏、重启期间关闭 Desktop 和草稿附件保留。
- [ ] 测试 Renderer 只保留当前和最近离开的 Session 展示投影，并在 OMP 恢复后按消息 ID 校正。
- [ ] 使用 v17.0.6 fixture 测试三项精确匹配；未知版本和格式变化回退为普通 `select`。
- [ ] 测试普通 `select` 有无 OMP 超时的两条路径，以及未知版本提示每次 Runtime 最多一次。
- [ ] 测试单项和批量允许或拒绝、默认焦点、Esc、5 行高度限制和内部滚动。
- [ ] 测试默认 30 秒、OMP 较短超时、100 毫秒收集窗口、窗口后台、系统睡眠和唤醒。
- [ ] 测试权限行单行省略、Content 排除、Workspace 内外路径、URL 脱敏、工作目录、多文件和未知工具回退。
- [ ] 测试新请求追加不抢焦点、不强制滚动、不重置截止时间；第二个请求加入时原区域平滑变为合并区。
- [ ] 测试单项立即回传、批量决定的请求快照、已处理状态、自动允许与手动允许区分。
- [ ] 测试 `cancel`、Stop、关闭 Desktop、重复 ID、迟到操作、Runtime 断开和用户操作与超时竞争。
- [ ] 测试 Renderer 重载恢复待确认列表和剩余时间，不重置倒计时或显示过期按钮。
- [ ] 测试确认区位于过程底部，到达时展开，用户折叠后摘要显示倒计时，自动允许后不重新展开。
- [ ] 测试完整审批 `title` 不进入 Renderer 快照、诊断日志或持久化数据。
- [ ] 测试“全部允许”模式仍显示 OMP 实际发出的安全确认或其他 `select`。

## 完成条件

- [ ] 每个 Session 可以保存独立权限，重启 Desktop 后仍正确。
- [ ] 新建、打开或切换 Session 后，OMP 实际启动参数与界面权限一致。
- [ ] 权限不同时能够无弹窗重启 Runtime 并恢复目标 Session，草稿和附件不丢失。
- [ ] v17.0.6 工具审批显示紧凑中文单项或合并确认区，并能单项或批量处理。
- [ ] 未处理工具审批在实际期限到达时自动允许，Stop、取消和 Runtime 断开不会误允许。
- [ ] 未知 OMP 版本安全回退为普通 Interaction，不执行 Desktop 自动允许。
- [ ] Renderer 重载后能恢复仍有效的审批请求和剩余时间。
- [ ] 权限配置、错误详情、Renderer 状态和诊断日志不泄露凭据或写入 Content。

## 复用重点

- MVP-02 的 Runtime Supervisor、15 秒 ready 超时、诊断日志和受控 IPC。
- MVP-04 的 Run、Action、Interaction 投影和过程折叠。
- MVP-05 的 Workspace ID、Session ID、删除、归档和最小 Desktop 配置。
- OMP v17.0.6 `extension_ui_request`、`respondExtensionUi`、`--approval-mode` 和审批标题格式。
- Radix UI Popover 和无障碍焦点管理。
