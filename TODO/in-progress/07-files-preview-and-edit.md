# MVP-07：文件树与上下文引用

- 状态：未开始
- 优先级：P1
- 前置任务：MVP-06
- 后续任务：MVP-08

## 目标

提供浏览 Workspace 和引用本地上下文所需的最小文件能力，但不在 MVP 内实现文件打开器或编辑器。

## 明确不做

- 不做调试器、插件市场和完整语言工作台。
- 不在启动时递归扫描整个 Workspace。
- 不实现复杂项目索引。
- 不实现应用内文件打开、文本预览、编辑和保存。
- 不引入 CodeMirror 等编辑器依赖。
- 不实现 Git Changes、Review 或 Diff。

## 任务清单

### 文件树

- [ ] 左侧第二栏显示文件树。
- [ ] 目录展开时再读取子项。
- [ ] 默认遵守 `.gitignore`。
- [ ] 支持刷新和文件系统变更合并。
- [ ] 支持按名称搜索文件。
- [ ] 对超大目录设置数量限制和明确提示。
- [ ] 文件树读取和搜索限制在当前 Workspace 内。

### 与对话联动

- [ ] 文件和目录可以加入 Prompt 上下文。
- [ ] Tool Call 修改文件后刷新对应节点。
- [ ] 只识别 Markdown 文件链接，以及反引号包裹且实际存在的本地路径。
- [ ] 普通点击不打开；`Ctrl+点击` 文件或目录时，由 Main 校验后交给系统文件管理器定位或打开。
- [ ] Workspace 外的现有路径允许定位，但 Renderer 不直接获得文件系统或 Electron Shell 权限。
- [ ] 不将完整文件树或完整仓库内容自动发送给模型。

### 测试

- [ ] 测试目录懒加载和 `.gitignore`。
- [ ] 测试路径越界和符号链接边界。
- [ ] 测试路径识别、存在性校验和 `Ctrl+点击` 系统调用。
- [ ] 测试普通文本中的类似路径不会被误识别。
- [ ] 测试 Tool Call 后的文件状态刷新。

## 完成条件

- [ ] 可以浏览当前 Workspace 文件。
- [ ] 可以搜索文件并将文件或目录加入对话上下文。
- [ ] 可以通过系统文件管理器定位有效路径。
- [ ] 文件树不会在启动时递归扫描完整项目。
- [ ] 大文件和大目录不会阻塞主线程。

## 复用重点

- 成熟的 `.gitignore` 匹配库：不要手写规则解析器。
- Electron Shell 只由 Main 通过受控 IPC 调用。
- Git CLI：MVP 不引入大型 Git SDK。
