## 我的明确选择

### 做 Code Agent Desktop

优先顺序：

1. **OpenCode Client/Server**
2. **Codex app-server**
3. **Pi Agent Core + 自己实现 TUI/GUI**
4. **Cline SDK**
5. **Goose API**

OpenCode 最开放。Codex app-server 的协议和执行模型更完整。Pi 最容易彻底改造，但你需要自己补大量产品层能力。 ([GitHub][1])

### 直接使用

- 终端日用：**Codex、OpenCode、Aider**
- 精致 TUI：**OpenCode、Pi、Oh My Pi、Crush**
- 桌面 GUI：**OpenCode Desktop、Goose、OpenWork、OpenHands**
- Claude Code 多模型路由：**Claude Code Router**
- 研究 Agent 架构：**Pi、Deep Agents、SWE-agent**

### 三个高星但不该放进主榜的项目

- **Roo Code**：已经停止运营并归档。
- **Vibe Kanban**：项目明确宣布正在 sunset。
- **Void**：维护工作暂停，Issues 和 PR 基本不再处理。 ([GitHub][21])

另外，Linux 上运行官方 Codex Desktop 的专项项目是 [ilysenko/codex-desktop-linux](https://github.com/ilysenko/codex-desktop-linux)，支持 `.deb`、`.rpm`、Arch 包、AppImage 和 Nix，约 1.1k Stars。它非常对题，但星数不足，因此没有计入前 20。 ([GitHub][22])
